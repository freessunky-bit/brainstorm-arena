## 2. AI 작업 행동 규칙 (예외 없이 적용)

### 2.1 복잡한 버그 수정 절차

복잡도가 높은 버그(여러 컴포넌트 연동, React 라이프사이클/리소스 관리, 동시성, 렌더링 파이프라인, 상태 동기화 등)를 수정할 때는 **곧바로 코드를 고치지 말 것.**

**필수 순서:**
1. 원인을 정확히 파악한 뒤 사용자에게 상세히 설명
2. 가능한 수정 방안 2~3가지를 제시
3. 사용자와 논의해 최종 방안을 합의한 뒤에만 실행

**추측 기반 즉흥 수정 절대 금지.**

### 2.2 fallback 분기 금지 원칙

근본 원인을 확인하지 않고 fallback 분기를 만들지 말 것.
- 데이터가 안 맞으면 → 데이터를 고친다
- 함수가 안 불리면 → 호출 경로를 추적해서 수정한다
- `|| 기본값` 폴백은 버그를 영원히 숨긴다 — 금지

**절대 금지 영역:** 결제·로그인·데이터 동기화 크리티컬 경로 / UI 렌더링 / 스탯 계산

**허용 예외:** OS·디바이스 차이에 의한 해상도·API 차이 등 진짜 예외 케이스에만 한정

### 2.3 한국어 원칙

- **모든 UI 텍스트**: 라벨, 플레이스홀더, 에러 메시지, 토스트 → 한국어
- **모든 AI 프롬프트**: 한국어 출력 명시 필수
- **JSON 응답**: 키는 영문 유지, 값은 한국어

### 2.4 코드 수정 시 필수 수행

1. `npm run build` 성공 확인
2. `src/App.jsx`의 `LAST_UPDATED` 상수를 현재 KST 시각으로 갱신 (형식: `"YYYY-MM-DD HH:MM KST"`)
3. `01_CHANGELOG.md`에 변경 내용 기록

## 3. 파일 구조 & 수정 가이드

### 수정 대상별 파일 매핑

| 수정 대상 | 파일 |
|----------|------|
| UI 컴포넌트 · 기능 로직 | `src/App.jsx` |
| AI 프롬프트 (시스템/유저) | `src/prompts.js` |
| API 호출 · 레이트 리미터 | `src/api.js` |
| 프로바이더 · 페르소나 · 상수 | `src/constants.js` |
| CSS 스타일 | `src/styles.js` |
| 로거 | `src/logger.js` |

### 주의사항

- `src/prompts.js`에 **모든 유저 프롬프트 빌더 함수**가 분리되어 있음 — 프롬프트 수정은 이 파일에서
- `FBO_FRAMEWORK` 변경 시 전체 분석 품질에 영향
- 아카이브 영역 `.page-enter` 래퍼에 `transform`이 `forwards`/`both`로 남으면 `position: fixed` 모달이 깨짐
